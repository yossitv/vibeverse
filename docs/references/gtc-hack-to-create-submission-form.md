# Hack to Create - Project Submission Form Reference

Source: https://airtable.com/appuMqcMKPSp1Eog8/pagqXe6ElIlXx6oa3/form

## Form Overview

**Title:** Hack to Create - Project Submission Form
**Instructions:** Complete this form to submit your project! Please review your demo video before submitting it!

## Required Fields (*)

| Field | Type | Notes |
|-------|------|-------|
| **Project Name** | Text | Required |
| **Submission Description** | Text | Required |
| **Demo Video** | URL/File | Required - review before submitting |
| **List Team Members Names and Emails** | Text | Required |
| **GitHub Repo** | URL | Required - must be public ("Make sure it's public so we can see your code :)") |
| **Models Used** | Text | Required |
| **Tools Used** | Text | Required |

## Optional Fields

| Field | Type | Notes |
|-------|------|-------|
| **What is the primary use case for your tool?** | Text | "Walk us through a typical workflow showing exactly how a creator would use your application to strike gold." |
| **Describe anything used from the NVIDIA AI ecosystem** | Text | "libraries, frameworks, recipes, etc" |

## Key Takeaways for Submission Strategy

- **Demo video is critical** - they emphasize reviewing it before submitting
- **GitHub repo must be public** - judges will review code
- **NVIDIA ecosystem usage** - dedicated field for this; using NVIDIA tools (NIM, CUDA, TensorRT, Omniverse, etc.) is clearly valued
- **Creator workflow** - the use case question wants a concrete walkthrough, not abstract description
- **Models + Tools** are separate required fields - be specific about what's used
